# discript-qa

**Универсальный QA-плагин для Claude Code.** Полноценный процесс тестирования приложений и сервисов с 10 специализированными агентами.

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
![Claude Code](https://img.shields.io/badge/Claude%20Code-plugin-purple)

---

## Что это

Плагин превращает Claude Code в полноценную QA-команду:

- **10 специализированных агентов** — функциональное, регрессионное, performance, security, a11y тестирование и другое
- **Единый формат багов** — все агенты выдают результаты в одном формате
- **Универсальность** — работает с любым стеком (web, API, mobile), настраивается под проект через бриф
- **Выборочный запуск** — не прогоняет всех скопом, ты выбираешь нужных для конкретной задачи
- **Встроенный workflow** — от начала аудита до исправлений и повторной проверки

## Быстрый старт

### 1. Установка как плагин (рекомендуется)

В Claude Code:

```
/plugin marketplace add ikobzar-git/discript-qa
/plugin install discript-qa@discript-qa
```

Или добавь в `~/.claude/settings.json`:

```json
{
  "extraKnownMarketplaces": {
    "discript-qa": {
      "source": {
        "source": "github",
        "repo": "ikobzar-git/discript-qa"
      }
    }
  },
  "enabledPlugins": {
    "discript-qa@discript-qa": true
  }
}
```

После установки перезапусти Claude Code.

### 2. Первое использование

В папке тестируемого проекта:

```
/qa-start         # заполнить бриф проекта
/qa-plan standard # выбрать агентов
/qa-run           # запустить прогон
/qa-report        # получить сводный отчёт
/qa-fix           # исправить найденные баги
/qa-retest        # повторно проверить
```

## Команды

| Команда | Назначение |
|---------|-----------|
| `/qa-start` | Заполнить бриф проекта (1 раз на проект) |
| `/qa-plan` | Умная рекомендация по набору агентов |
| `/qa-run` | Запустить выбранных агентов параллельно |
| `/qa-report` | Сводный отчёт с приоритизацией |
| `/qa-fix` | Исправить найденные баги |
| `/qa-retest` | Проверить фиксы и поймать регрессии |
| `/qa-reproduce` | Воспроизвести баг по описанию пользователя |
| `/qa-help` | Справка по всем командам |

## Агенты

### Аудит
- **qa-functional** — проверка работоспособности заявленного функционала (happy path)
- **qa-regression** — проверка, что ранее работавшее не сломалось
- **qa-consistency** — поиск непоследовательностей UX во всём продукте
- **qa-edge-cases** — граничные случаи, нестандартный ввод, редкие сценарии
- **qa-integration** — стыки систем, API, webhook-и, внешние сервисы
- **qa-performance** — Core Web Vitals, Lighthouse, скорость API
- **qa-accessibility** — WCAG 2.1 AA, скрин-ридеры, клавиатурная навигация
- **qa-security** — OWASP Top 10, уязвимости

### Расследование
- **bug-reproducer** — минимальный воспроизводимый сценарий по размытому описанию

### Исправление
- **developer** — фиксы найденных багов по приоритету

## Пресеты

Для типовых задач:

| Пресет | Агенты | Когда использовать |
|--------|--------|--------------------|
| `smoke` | functional + edge-cases | После мелких правок |
| `standard` | functional + consistency + edge-cases + a11y | Обычный аудит |
| `deep` | Все, кроме security | Перед важным релизом |
| `release` | Все 10 | Перед прод-релизом |
| `security` | security + integration | Security-аудит |
| `performance` | performance + a11y | Оптимизация метрик |
| `regression` | regression + functional | После крупных изменений |
| `bug-hunt` | reproducer + functional + edge-cases | Жалоба от пользователя |

Использование:
```
/qa-run smoke
/qa-run release
```

## Структура файлов проекта

После `/qa-start` в тестируемом проекте создаётся:

```
<проект>/
├── QA_BRIEF.md              # бриф проекта (заполняется 1 раз)
└── .qa/
    ├── bugs/                # карточки багов по агентам
    ├── screenshots/         # evidence
    ├── reports/             # сводные отчёты и планы прогонов
    ├── regression-registry.json
    └── fixtures/            # тестовые данные
```

## Единый формат бага

Все агенты выдают находки в одинаковом формате:

```markdown
### [Название]
- **Серьёзность:** БЛОКЕР | КРИТИЧНО | ВАЖНО | МЕЛКИЙ
- **Категория:** функционал | UX | a11y | performance | security | ...
- **Компонент:** (страница/модуль)

**Шаги воспроизведения:**
1. ...

**Ожидаемое / Фактическое:**
...

**Рекомендация:**
...
```

Полный шаблон: [templates/BUG_REPORT.md](templates/BUG_REPORT.md)

## Workflow

```
┌─────────────────┐
│  /qa-start      │ ← заполняем бриф (1 раз)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  /qa-plan       │ ← выбираем агентов
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  /qa-run        │ ← параллельный прогон агентов
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  /qa-report     │ ← сводный отчёт
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  /qa-fix        │ ← исправление
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  /qa-retest     │ ← повторная проверка
└────────┬────────┘
         │
         ▼
   0 БЛОКЕРОВ
   0 КРИТИЧНЫХ
         │
         ▼
      ✅ релиз
```

**Правило 2 итераций:** максимум 2 полных цикла аудит→фикс→retest. Если после 2-го остались блокеры — сигнал о более глубокой проблеме, требующей обсуждения.

## Требования

- **Claude Code** (любая актуальная версия)
- Для live-тестирования: **Playwright MCP** (опционально, но рекомендуется)
- Для a11y: **axe-core** (подтягивается через Playwright)
- Для performance: **Lighthouse CLI** (опционально)

Если инструменты недоступны — агенты работают в режиме статического анализа кода.

---

## Настройка под свой стек

В `QA_BRIEF.md` можно переопределить инструменты:

```markdown
## 7. Инструменты тестирования
- E2E-браузер: Cypress
- A11y: WAVE
- Performance: WebPageTest
- API: Postman CLI (newman)
```

Агенты адаптируются под указанные инструменты.

## Как делиться с командой

1. Каждый член команды устанавливает плагин через `/plugin marketplace add ikobzar-git/discript-qa` — один раз.
2. Плагин доступен во всех проектах автоматически.
3. В каждом проекте, где нужен QA, выполняется `/qa-start` — заполняется свой `QA_BRIEF.md`.
4. Обновления плагина приходят автоматически при релизах.
