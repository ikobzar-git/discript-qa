# Чек-лист: мобильная версия

## A. Viewport и масштабирование

- [ ] `<meta name="viewport" content="width=device-width, initial-scale=1">` установлен
- [ ] Нет `user-scalable=no` или `maximum-scale=1` (блокирует зум — нарушение a11y)
- [ ] Страница корректно адаптируется от 320px до 1920px
- [ ] Нет горизонтального скролла на любых размерах
- [ ] Не ломается в landscape-ориентации
- [ ] При повороте экрана данные формы не теряются

## B. Размеры элементов

### Touch targets
- [ ] Все интерактивные элементы ≥ **44x44px** (Apple HIG) или ≥ **48x48px** (Material)
- [ ] Между тап-целями есть зазор (≥8px)
- [ ] Маленькие ссылки в тексте — имеют увеличенную зону тапа
- [ ] Клик по чекбоксу работает при тапе на label (не только на квадрат)

### Текст
- [ ] Body text ≥ 16px (меньше — сложно читать)
- [ ] Заголовки имеют адекватные размеры
- [ ] Межстрочный интервал ≥ 1.5 для body
- [ ] Нет текста на изображениях без альтернативы

## C. Навигация

### Меню
- [ ] Гамбургер-меню открывается по тапу
- [ ] Меню закрывается по тапу вне области
- [ ] Меню закрывается по ESC (если есть клавиатура)
- [ ] Меню закрывается свайпом (опционально)
- [ ] Активный пункт меню выделен
- [ ] Подменю открываются корректно

### Нижняя навигация (если есть)
- [ ] Закреплена снизу
- [ ] Не перекрывается клавиатурой
- [ ] Иконки + текст (или только иконки с aria-label)
- [ ] Активная вкладка выделена

### Breadcrumbs
- [ ] На мобилке — либо скрыты, либо компактный вариант
- [ ] Не ломают layout

## D. Формы на мобилке

### Клавиатура
- [ ] При фокусе на email → email-клавиатура (`inputmode="email"`)
- [ ] При фокусе на phone → numpad (`inputmode="tel"`)
- [ ] При фокусе на число → numeric keyboard
- [ ] При фокусе на URL → URL keyboard
- [ ] Автокапитализация для имён (`autocapitalize="words"`)
- [ ] Без автокапитализации для email/login (`autocapitalize="off"`)

### Поведение
- [ ] При фокусе поле не уезжает под клавиатуру
- [ ] Submit-кнопка не закрыта клавиатурой
- [ ] Next/Previous на клавиатуре переключает поля
- [ ] Enter на последнем поле → submit
- [ ] Autofill из менеджера паролей работает

### Специфика
- [ ] Date picker — нативный (не кастомный)
- [ ] File upload — открывает камеру/галерею
- [ ] Select — нативный picker
- [ ] Textarea адекватно растёт по содержимому

## E. Контент

### Изображения
- [ ] Грузятся в мобильном разрешении (responsive images)
- [ ] `srcset` / `<picture>` используется
- [ ] Lazy-load для картинок ниже viewport
- [ ] Нет огромных картинок на мобилке

### Таблицы
- [ ] Горизонтальный скролл в контейнере, не всей страницы
- [ ] Sticky-заголовок (опционально)
- [ ] Или альтернатива — карточный вид на мобилке

### Длинный текст
- [ ] "Читать далее" / аккордеоны для длинного контента
- [ ] Нет мелкого текста (<14px)
- [ ] Ссылки в тексте легко тапаются

## F. Производительность на мобилке

### Загрузка
- [ ] LCP < 2.5s на 3G Slow
- [ ] FCP < 1.8s
- [ ] Интерактивность < 3.5s
- [ ] Bundle < 300KB gzipped для главного чанка
- [ ] Нет render-blocking JS

### Runtime
- [ ] Скролл 60fps
- [ ] Анимации без дропов
- [ ] Нет long tasks >50ms
- [ ] Память не течёт (тест на долгое использование)

## G. Специфика iOS Safari

- [ ] `-webkit-overflow-scrolling: touch` для smooth scroll в скроллящихся областях
- [ ] Safe areas — поддержка `env(safe-area-inset-*)` для iPhone с челкой
- [ ] Нет 100vh без учёта адресной строки (использовать `dvh` или JS)
- [ ] Клик на кнопке внутри label работает (старый баг iOS)
- [ ] Date input работает как нативный picker
- [ ] Нет double-tap zoom на кнопках (`touch-action: manipulation`)

## H. Специфика Android Chrome

- [ ] Auto-fill работает корректно
- [ ] Шрифты загружаются вовремя (нет FOIT)
- [ ] PWA install prompt (если применимо)

## I. Responsive breakpoints

Проверь на ключевых размерах:
- [ ] 320px (маленькие мобильные — iPhone SE)
- [ ] 375px (стандартные iPhone)
- [ ] 414px (iPhone Plus)
- [ ] 768px (планшеты portrait / большие мобильные)
- [ ] 1024px (планшеты landscape)
- [ ] 1280px и выше (desktop)

## J. Оффлайн и нестабильная сеть

- [ ] При потере сети → понятное сообщение
- [ ] Service Worker кэширует критичное (если PWA)
- [ ] Повторная попытка при восстановлении сети
- [ ] Не теряются данные формы при кратком отключении
- [ ] Нет бесконечных спиннеров

## K. PWA-специфика (если применимо)

- [ ] Manifest.json присутствует
- [ ] Иконки 192x192 и 512x512
- [ ] Theme color настроен
- [ ] Устанавливается как PWA
- [ ] Работает offline (базовая страница)
- [ ] Push-уведомления работают

## L. Доступность на мобилке

- [ ] VoiceOver (iOS) озвучивает всё корректно
- [ ] TalkBack (Android) тоже
- [ ] Swipe-навигация работает
- [ ] Zoom до 200% не ломает layout
- [ ] Prefers-reduced-motion уважается
