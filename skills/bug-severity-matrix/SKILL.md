# Матрица серьёзности багов

## 4 уровня

### БЛОКЕР
Продукт непригоден к использованию. Нельзя выпускать в прод.

**Критерии (хотя бы один):**
- Невозможно зарегистрироваться / войти
- Не работает ключевой сценарий, ради которого продукт существует (оформить заказ, оплатить, отправить сообщение)
- Утечка приватных данных (чужие заказы, чужие email, токены в URL)
- Потеря денег клиента или бизнеса
- Продукт падает / показывает 500 / белый экран на критичных страницах
- Невозможно установить / запустить продукт

**SLA на фикс:** немедленно, hot-fix.

---

### КРИТИЧНО
Продукт работает, но с серьёзными проблемами, ощутимо бьющими по бизнес-метрикам.

**Критерии:**
- Основной сценарий работает нестабильно (платёж проходит со 2-3 раза)
- Ломается в популярном браузере / на мобилке
- Значительное замедление (LCP > 5s)
- Уязвимость, эксплуатация которой ВОЗМОЖНА, но требует нестандартных условий (некритичный XSS, IDOR на неважных данных)
- Не работает важный, но не главный сценарий (восстановление пароля, уведомления)
- A11y-нарушение, блокирующее использование для >5% пользователей

**SLA на фикс:** до ближайшего релиза.

---

### ВАЖНО
Заметные проблемы, но есть обходной путь. Не блокирует основное использование.

**Критерии:**
- Непоследовательность UX (разные модалки закрываются по-разному)
- Некорректные сообщения об ошибках (малоинформативны)
- Визуальные баги вне ключевых зон
- Проблемы в редких сценариях (экзотические edge-cases)
- Низкий Lighthouse Score в некритичных разделах
- A11y на AA уровне в некритичных местах

**SLA на фикс:** в ближайший спринт.

---

### МЕЛКИЙ
Косметика, доработки, улучшения.

**Критерии:**
- Небольшие визуальные несоответствия (1-2px отступ)
- Опечатки в текстах
- Мелкие улучшения UX
- Дубли логов
- TODO в коде

**SLA на фикс:** когда дойдут руки, по возможности.

## Решающие факторы при выборе уровня

### 1. Влияние на бизнес
- Потеря денег / конверсии → БЛОКЕР / КРИТИЧНО
- Репутационный риск → КРИТИЧНО
- Только внутренние пользователи → на уровень ниже

### 2. Количество затронутых пользователей
- 100% пользователей → повысить на уровень
- <1% (только IE11, только Windows XP) → понизить на уровень

### 3. Частота воспроизведения
- Каждый раз → как есть
- Иногда (50%) → можно понизить, если impact не критичный
- Редко (<5%) → понизить, но добавить мониторинг

### 4. Обходной путь
- Есть простой обходной путь → можно понизить
- Нет обходного пути → как есть или выше

### 5. Юридические требования
- Нарушение GDPR / 152-ФЗ / PCI-DSS → БЛОКЕР или КРИТИЧНО
- Нарушение брендбука без юр.последствий → ВАЖНО или МЕЛКИЙ

## Типовые ошибки оценки

**Завышение уровня:**
- "Неудобная кнопка" → это не КРИТИЧНО, это ВАЖНО или МЕЛКИЙ
- "Не нравится цвет" → это не баг, это замечание к дизайну

**Занижение уровня:**
- "Воспроизводится редко" → но каждый раз ТЕРЯЕТСЯ ЗАКАЗ → БЛОКЕР
- "Только на Safari" → Safari = 20% рынка, это не "edge case"

## Шаблон оценки

Перед назначением уровня спроси себя:
1. Может ли пользователь закончить то, ради чего пришёл? (нет → БЛОКЕР)
2. Потеряют ли пользователи или бизнес деньги? (да → БЛОКЕР / КРИТИЧНО)
3. Большая ли доля пользователей затронута? (>10% → повысить)
4. Есть ли альтернативный путь? (нет → повысить)
5. Можно ли откатить? (нет и уже в проде → БЛОКЕР даже если "мелочь")
