# /qa-reproduce — воспроизведение бага

Команда запускает `bug-reproducer` для получения минимального воспроизводимого сценария по сырому описанию проблемы.

## Варианты использования

```
/qa-reproduce                        # интерактивный режим — опиши проблему
/qa-reproduce "не работает кнопка оплаты"    # с описанием сразу
/qa-reproduce --sentry-url=...       # импорт из Sentry
```

## Что делает команда

1. Получает описание проблемы (от пользователя, из Sentry, из тикета).
2. Запускает `bug-reproducer` агента.
3. Агент пытается воспроизвести и минимизировать сценарий.
4. На выходе — полная карточка бага в формате `BUG_REPORT.md`.
5. Карточка сохраняется в `.qa/bugs/reproduced/`.
6. Пользователю предлагается передать её в `/qa-fix` или исследовать дальше.

## Инструкции для Claude

### Алгоритм

1. **Получи исходное описание проблемы:**
   - Если пользователь передал текст в аргументах — используй его
   - Если в аргументах URL Sentry — fetch ошибки через API
   - Иначе — спроси пользователя:
     ```
     Опиши проблему настолько подробно, насколько знаешь:
     - Что делал пользователь?
     - Что ожидал?
     - Что произошло?
     - Дополнительная инфа: браузер, устройство, время?
     ```

2. **Передай запрос** в bug-reproducer:

```
Task(
  description: "Reproduce user-reported bug",
  subagent_type: "bug-reproducer",
  prompt: """
  Воспроизведи и минимизируй следующий баг:
  
  === Исходное описание ===
  {исходный текст от пользователя}
  
  === Контекст ===
  QA_BRIEF.md: [путь]
  Staging URL: [из брифа]
  
  === Задача ===
  1. Попытайся воспроизвести
  2. Минимизируй сценарий
  3. Определи серьёзность
  4. Сохрани карточку в .qa/bugs/reproduced/B-{NNN}.md
  5. Верни краткое резюме: удалось ли воспроизвести, какая серьёзность
  """
)
```

3. **После завершения** bug-reproducer:
   - Если удалось воспроизвести:
```
✅ Баг воспроизведён: B-042

**Название:** Оплата не проходит при сумме >10000₽
**Серьёзность:** БЛОКЕР
**Повторяемость:** 3/3 (100%)
**Компонент:** /checkout

Минимальные шаги:
1. Открыть /checkout в Chrome
2. Добавить товар >10000₽
3. Нажать "Оплатить"
→ Кнопка не реагирует, TypeError в консоли

Полная карточка: .qa/bugs/reproduced/B-042.md

Следующие шаги:
- /qa-fix B-042 — передать на исправление
- /qa-run functional — проверить, нет ли других проявлений
```

   - Если НЕ удалось воспроизвести:
```
⚠️ Баг не воспроизведён

Что пробовал агент:
- (список попыток)

Для дальнейшего расследования нужна дополнительная информация:
- Точный URL, где произошла ошибка
- Время события (чтобы поискать в логах)
- User-Agent пользователя
- Скриншот / видео с клиента

Передать эти вопросы пользователю для уточнения?
```

### Важно

- **Не гадай.** Если воспроизвести не удалось — не фабрикуй сценарий. Попроси уточнений.
- **Работай только на staging или локально.** Если доступен только production — предупреди о рисках.
- **Сохраняй evidence.** Скриншоты, видео, логи складывай в `.qa/screenshots/` и ссылайся в карточке.
