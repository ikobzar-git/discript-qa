# /qa-help — справка

Показывает полный список команд плагина с описанием и типовым применением.

## Инструкции для Claude

Выведи следующий справочник:

```
# 📋 discript-qa — справка

Универсальный QA-процесс для Claude Code.

## Команды

### Настройка проекта
- /qa-start          — заполнить бриф проекта (1 раз на проект)

### Планирование
- /qa-plan           — умная рекомендация по набору агентов
- /qa-plan smoke     — быстрый пресет
- /qa-plan standard  — обычный аудит
- /qa-plan release   — полный прогон перед релизом

### Выполнение
- /qa-run            — запуск агентов из плана
- /qa-run smoke      — быстрый прогон (functional + edge-cases)
- /qa-run all        — все 10 агентов
- /qa-run a11y,perf  — произвольный набор

### Отчёт
- /qa-report         — сводный отчёт с приоритизацией

### Исправление
- /qa-fix            — фиксы по блокерам и критичным
- /qa-fix all        — фикс всех багов
- /qa-fix B-001      — конкретный баг
- /qa-fix --dry-run  — план без применения

### Повторная проверка
- /qa-retest         — проверить фиксы + регрессия
- /qa-retest full    — полный повторный прогон

### Воспроизведение
- /qa-reproduce      — воспроизвести баг от пользователя
- /qa-reproduce "описание"  — с описанием сразу

## Типовые сценарии

### Первый аудит нового проекта
```
/qa-start            # заполнить бриф
/qa-plan standard    # выбрать агентов
/qa-run              # запуск
/qa-report           # отчёт
/qa-fix              # фиксы
/qa-retest           # проверка фиксов
```

### Жалоба от пользователя
```
/qa-reproduce "описание проблемы"
/qa-fix B-NNN
/qa-retest B-NNN
```

### Проверка перед релизом
```
/qa-plan release     # все агенты
/qa-run
/qa-report
# разбираем все БЛОКЕРЫ и КРИТИЧНЫЕ
/qa-fix critical
/qa-retest full
```

### Быстрая проверка после мелких правок
```
/qa-run smoke
/qa-report
```

## Агенты

Запускаются командами /qa-plan и /qa-run.

| Агент            | Назначение |
|------------------|-----------|
| qa-functional    | Работает ли заявленный функционал |
| qa-regression    | Не сломалось ли ранее работавшее |
| qa-consistency   | Однородность поведения в продукте |
| qa-edge-cases    | Граничные случаи и нестандартный ввод |
| qa-integration   | Стыки систем, API, webhook-и, внешние сервисы |
| qa-performance   | Скорость, метрики Core Web Vitals |
| qa-accessibility | Доступность, WCAG 2.1 AA |
| qa-security      | Уязвимости по OWASP Top 10 |
| bug-reproducer   | Воспроизведение багов от пользователей |
| developer        | Исправление найденных багов |

## Файловая структура проекта

После /qa-start создаются:

```
<проект>/
├── QA_BRIEF.md              # бриф проекта (заполняется 1 раз)
└── .qa/
    ├── bugs/                # карточки багов
    │   ├── functional/
    │   ├── edge-cases/
    │   ├── consistency/
    │   └── ...
    ├── screenshots/         # evidence
    ├── reports/             # сводные отчёты
    │   ├── plan-*.md
    │   └── QA_REPORT_*.md
    ├── regression-registry.json
    └── fixtures/            # тестовые данные
