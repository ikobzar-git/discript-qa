# /qa-report — сводный отчёт

Команда собирает все баги из `.qa/bugs/`, группирует, приоритизирует и формирует итоговый отчёт.

## Что делает команда

1. Читает все карточки багов из `.qa/bugs/*/`.
2. Читает `QA_BRIEF.md` для контекста.
3. Агрегирует по серьёзности, категориям, агентам.
4. Выявляет возможные конфликты между агентами (одно и то же замечание с разных сторон).
5. Формирует отчёт по шаблону `FINAL_REPORT.md`.
6. Сохраняет в `.qa/reports/QA_REPORT_{YYYY-MM-DD}.md`.
7. Выводит краткую версию в чат.

## Инструкции для Claude

### Алгоритм

1. **Собери все баги:**
   ```bash
   find .qa/bugs -name "*.md" -type f
   ```
2. **Парси каждую карточку** — извлекай поля (серьёзность, категория, компонент, кто нашёл).

3. **Объедини дубли.** Если два агента нашли один баг с разных сторон — слепи в один, перечислив всех findings.

4. **Детекция конфликтов** между рекомендациями:
   - Один агент говорит "добавь X"
   - Другой говорит "убери X"
   → Помечай как **КОНФЛИКТ**, требующий решения.

5. **Посчитай метрики** (если собирались):
   - Lighthouse scores
   - Количество проверенных сценариев
   - % покрытия критичных фич

6. **Сгенерируй отчёт** в двух форматах:
   
   a) **Полный** (в файл `.qa/reports/QA_REPORT_{дата}.md`) — по шаблону `FINAL_REPORT.md`
   
   b) **Краткий** (в чат) — следующего вида:

```
## 📋 QA-отчёт по проекту {название}

**Дата:** 2026-04-17
**Итерация:** 1 из 2
**Прогнали:** qa-functional, qa-edge-cases, qa-consistency, qa-accessibility

### Статистика

| Серьёзность | Найдено |
|-------------|---------|
| 🔴 БЛОКЕР    | 1       |
| 🟠 КРИТИЧНО  | 4       |
| 🟡 ВАЖНО     | 9       |
| 🟢 МЕЛКИЙ    | 4       |
| **Итого**   | **18**  |

### 🔴 БЛОКЕРЫ (требуют немедленного внимания)

**1. Не работает оплата заказов свыше 10000₽**
- Компонент: /checkout
- Категория: функционал
- Кто нашёл: qa-functional
- Кратко: TypeError при формировании суммы → кнопка не реагирует
- Файл: `.qa/bugs/functional/B-001.md`

### 🟠 КРИТИЧНЫЕ (4)

(короткий список: название, компонент, категория)

### 🟡 ВАЖНЫЕ (9)

(короткий список)

### 🟢 МЕЛКИЕ (4)

(свёрнутый список)

### ⚠️ Конфликты рекомендаций

Нет конфликтов. / Обнаружено N конфликтов — см. отчёт.

### Статус приёмки

### Рекомендация

Полный отчёт сохранён: .qa/reports/QA_REPORT_2026-04-17.md
```

### Важно

- **Отчёт читают и разработчики, и не-технические люди.** Пиши ясно, без жаргона.
- **Группируй связанные баги.** Если 5 багов про формы — объедини в один кластер с подпунктами.
- **Сортируй по серьёзности внутри категорий.**
- **Не опускай мелкие баги**, но сверни их — они не должны перегружать отчёт.
- **Всегда заканчивай рекомендацией**: что делать дальше.

### Если багов 0

```
## ✨ QA-отчёт

Найдено багов: **0**

Все {N} агентов прошли чисто. Приёмка пройдена.

Что проверено:
- ... (список фич)

Рекомендация: финальный прогон не требуется.
```
